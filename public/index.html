ADDENDUM:

Setting Up ExpressJS Server: I created an ExpressJS server script named `server.js` to handle HTTP requests. This script includes middleware setup, such as `express.urlencoded` for parsing form data and `morgan` for logging. Additionally, it defines a route (`/do_a_random`) to generate a random number and serves static files from the `public` directory.

Installing Necessary npm Packages: To utilize ExpressJS and enhance logging capabilities, I installed three npm packages: `express`, `morgan`, and `forever`. These packages provide functionalities for creating the server, improving logging, and managing server processes, respectively.

Structuring Website Directory: I organized the project directory by creating a `public` folder to host static files, including HTML and CSS files for each lab. These files are served by ExpressJS.

Running the Express Server: To run the Express server locally for development, I executed the command `node server.js local` in the terminal/command prompt within the website directory. This command starts the server on port 8080.

Testing the Server and Form Submission: I verified that the server was working correctly by visiting the specified URLs (`http://localhost:8080/itc505/lab-4` and `http://localhost:8080/do_a_random`). Then, I created a new webpage.

Implementing Form Submission Handling: I created a form in `index.html` with input fields for adjective, noun, verb, adverb, and noun2. Upon form submission, ExpressJS handles the POST request to `/submit`. The server extracts the form data and constructs a Mad Libs story using string interpolation. If any field is missing, it returns an error message.
